MINER DEVICE API INTEGRATIONS
================================================================================

This software integrates with mining hardware devices using their publicly
documented APIs. These integrations are provided for user convenience to
manage devices that the user owns and has authorization to access.

IMPORTANT: These integrations use ONLY publicly documented APIs provided
by the device manufacturers. No reverse engineering or proprietary
protocols are used.

A. AxeOS / ESP-Miner Based Devices
   Devices: BitAxe (Ultra, Supra, Hex, Gamma, GT), NMAxe, NerdQAxe,
            NerdOctaxe Gamma, Lucky Miner (LV06/LV07/LV08/LG07),
            Jingle Miner (BTC Solo Lite/Pro), Zyber (8S/8G/8GP),
            Hammer Miner, GekkoScience (Compac/NewPac/R606)
   Protocol: HTTP REST API (Port 80)
   Source: Open-source ESP-Miner firmware (GPL-3.0)
   GitHub: https://github.com/skot/ESP-Miner
   Documentation: Publicly available in firmware source code

   Note: Spiral Pool communicates with these devices via standard HTTP REST
   API calls only. No ESP-Miner source code is included, referenced, or
   compiled into Spiral Pool. The API is a network interface, not a code
   dependency. Some ESP32-based miners (NerdMiner, HAN SOLOminer, BitMaker,
   SparkMiner) do not have an HTTP API; stats for such devices are obtained
   from pool connections only.

B. BraiinsOS (BOS+) Firmware
   Devices: Antminers running BraiinsOS firmware
   Protocol: HTTP REST API (Port 80, documented at /api/v1/)
   Documentation: https://developer.braiins-os.com/
   License: Proprietary firmware with publicly documented API

C. Vnish Firmware
   Devices: Antminers running Vnish custom firmware
   Protocol: HTTP REST API (Port 80) + CGMiner RPC (Port 4028)
   Documentation: https://vnish.group/en/faq-vnish-en/
   License: Proprietary firmware with publicly documented API

D. LuxOS Firmware
   Devices: Antminers running LuxOS firmware
   Protocol: TCP Socket API (Port 4028)
   Documentation: https://docs.luxor.tech/firmware/api/intro
   License: Proprietary firmware with publicly documented API
   Python Package: luxos (MIT License)

E. CGMiner API (Standard Mining Protocol)
   Devices: Avalon, Canaan AvalonMiner, Antminer, Whatsminer, Innosilicon,
            GekkoScience, iPollo, Ebang/Ebit, ePIC, FutureBit, Elphapex,
            iBeLink, VolcMiner
   Protocol: TCP Socket API (Port 4028)
   Documentation: https://github.com/ckolivas/cgminer/blob/master/API-README
   License: CGMiner is GPL-3.0 licensed; API protocol is open standard

F. Goldshell Miners
   Devices: Goldshell Mini DOGE series, LT5, LT6, LT Lite, DG Max, BYTE
   Protocol: HTTP REST API (Port 80)
   Documentation: Publicly accessible via device web interface

G. ePIC Blockchain Miners
   Devices: ePIC BlockMiner series
   Protocol: HTTP REST API (Port 4028)
   Documentation: https://epicblockchain.io/support/
   GitHub: https://github.com/epicblockchain/epic-dashboard

H. iPollo Miners
   Devices: iPollo V1, V1 Mini, G1, etc.
   Protocol: HTTP + CGMiner API (Port 4028)
   Documentation: https://ipollo.com/pages/v1-server-setup-manual

I. Whatsminer BTMiner API (MicroBT Proprietary)
   Devices: Whatsminer M30S through M79 series
   Protocol: CGMiner API (Port 4028) + proprietary BTMiner extensions
   Documentation: Whatsminer API v2.0.3 (official documentation)
   License: Proprietary firmware with publicly documented API

J. Bitdeer / Sealminer
   Devices: Sealminer A2, A2 Pro, A2 Hydro, A3, A3 Pro, A3 Hydro series
   Protocol: Stratum protocol (user-agent detection only)
   Documentation: N/A (user-agent classification only, no API integration)
   Note: Spiral Pool classifies Sealminer devices by user-agent string only.
   No direct device API integration is implemented.

K. Auradine / Teraflux
   Devices: Teraflux AT2880, AI3680, AH3880
   Protocol: Stratum protocol (user-agent detection only)
   Documentation: N/A (user-agent classification only, no API integration)
   Note: Spiral Pool classifies Auradine devices by user-agent string only.
   No direct device API integration is implemented.

L. Elphapex Miners (Scrypt)
   Devices: DG Home, DG1, DG1+, DG1 Lite, DG2, DG2+, DG2 Mini, DG Hydro
   Protocol: CGMiner API (Port 4028) / Stratum user-agent detection
   Documentation: Publicly documented CGMiner API

M. VolcMiner (Scrypt)
   Devices: D1 Mini, D1, D1 Pro, D1 Lite, D1 Hydro, D3
   Protocol: Proprietary firmware with web GUI / Stratum user-agent detection
   Documentation: Device web interface

N. FluMiner (SHA-256d + Scrypt)
   Devices: T3 (SHA-256d), L1, L1 Pro, L2, L3 (Scrypt)
   Manufacturer: Flu Electronic Technology (Hong Kong) Co., Ltd.
   Protocol: Proprietary firmware with web UI / Stratum user-agent detection
   Documentation: Device web interface

O. iBeLink (Scrypt)
   Devices: BM-L3
   Protocol: CGMiner API (Port 4028) / Stratum user-agent detection
   Documentation: Device web interface

P. User-Agent-Only Classifications (No API Integration)
   The following devices are classified by user-agent string for difficulty
   assignment purposes only. Spiral Pool does not connect to these devices
   or use any device API:
   - Lucky Miner (LV06, LV07, LV08, LG07)
   - Jingle Miner (BTC Solo Lite, BTC Solo Pro)
   - Hammer Miner / PlebSource (Doge Digger)
   - HAN SOLOminer, BitMaker, SparkMiner (ESP32 variants)
   - NiceHash, MiningRigRentals, Cudo Miner (hashrate rental services)
   - Zergpool, Prohashing, Mining Dutch, Zpool, WoolyPooly,
     HeroMiners, unMineable (mining pool proxies)

   GDPR NOTICE: Loading fonts from Google Fonts CDN causes end-user browsers
   to make requests to Google servers, which transmits the user's IP address
   to Google. The Regional Court of Munich (LG München, Case No. 3 O 17493/20,
   January 20, 2022) ruled that dynamically loading Google Fonts without consent
   can violate GDPR, as it transmits IP addresses to Google (a US company)
   without adequate legal basis.

   OPERATOR RECOMMENDATIONS:
   - For GDPR-compliant deployments, consider self-hosting the font files
     instead of loading from Google's CDN
   - To self-host: download font files from fonts.google.com, serve from your
     own infrastructure, and update CSS @import URLs accordingly
   - If continuing to use Google Fonts CDN, implement a cookie/consent banner
     for dashboard users informing them of the data transfer
   - The font files themselves are licensed under SIL OFL 1.1, which permits
     redistribution and self-hosting

   Note: The fonts are used only in the web dashboard UI. Stratum mining
   connections (the primary pool function) do not involve any font loading
   or Google service connections.
